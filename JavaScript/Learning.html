JavaScript Let

let fue introducido en ES6 en 2015

Principalmente lo que debes saber de let es:

  • Las variables definidas con let no pueden ser redeclarada
  • Las variables definidas con let siempre deben ser declaradas antes de ser usadas generando asi buenas practicas
  • Las variables definidas con let tienen enfoque dentro de bloques{}

No pueden ser redeclaradas

lo que significa que esto es un error

      let z = "Manuel";
      let z = 2;

Enfoque de bloques

Lo que significa que variables declaradas con let dentro de un bloque solo pueden ser accedidas dentro del mismo bloque y no afuera de este 

      // no se puede usar aqui
      {
        // se puede usar aqui
        let $$ = 24;
      }

Elevacion de Let

Las variables definidas con let son elevadas al inicio del bloque, pero no son inicializadas 

        nombreAuto = 'Mazda';
        let nombreAuto = 'Nissan';
        // esto tirara un error de referencia

Diferencias con var

  • Variables declaradas con var pueden ser redeclaradas
  • Variables declaradas con var no tienen enfoque de bloques, lo que significa que se puede acceder a ellas fuera de un bloque de codigo lo que tambien significa que si declaramos una variable con var fuera de un bloque, y luego la redeclaramos dentro de un bloque, este cambio afectara tambien fuera del bloque
  • Una ventaja de var es que cuando declaramos con var, se eleva automaticamente al inicio del bloque lo que significa que podemos usar una variable antes de declararla y puede ser inicializada en cualquier momento

JavaScript Const

Al igual que let, const fue introducido en ES6

sus principales caracteristicas son:

  • Su valor no puede ser redeclarado
  • Su valor no puede ser reasignado
  • Tiene enfoque de bloques

No se puede reasignar

Es decir que el valor que de esta no puede ser reasignado mas tarde en el codigo 

        const e = 2.8787;
        e = 2.9;
        // esto tirara un error

Siempre se el debe asignar un valor al declararlo

        const PI = 3; // Forma correcta
    
        const PI;
        PI = 3.14;

la palabra const no define un valor constante, define una referencia constante a un valor

  • Cambiar los elementos de un objeto constante

Funciones

Una funcion es un bloque de codigo que ejecuta una accion en particular

Una funcion se ejecuta cuando algo la llama(invoca)

      function saludo(){
        console.log("HOLA AMIGOS");
      }

Sintaxis de las funciones

Una funcion se define con la palabra reservada function, seguido del nombre de la funcion y despues parentesis y luego llaves donde se pondra el codigo a ejecutarse

Los nombres de las funciones siguen las mismas reglas que las variables

Los parentesis pueden contener parametros separados por comas

      function nombre(parametro1, parametro2, parametro3){
        // codigo a ser ejecutado
      }

Los parametros son listados como se dijo, dentro de los parentesis

Los argumentos son los valores recibidos por la funcion cuando es invocada

Dentro de la funcion, los argumentos(parametros) se comportan como variables locales

Invocacion de funciones

Como ya se dijo, una funcion ejecuta el codigo dentro de ella, cuando algo la invoca o llama a la funcion

Algunos casos de esta invocacion son:

  • Cuando ocurre un evento(por ejemplo cuando se presiona un boton)
  • Cuando se la llama desde el mismo codigo de JavaScript
  • Automaticament(Cuando se auto invoca)

Retornables de una funcion

Cuando JavaScript alcance a una declaracion de la palabra reservada return, entonces la funcion se dejara de ejecutar

Si la funcion fue invocada desde una declaracion, JavaScript retornara a ejecutar el codigo despues de declaracion invocadora

Las funciones normalmente computan un valor para retornar, el valor es retornado o devuelto al llamador , es decir, devuelve el valor computado a ese algo que lo llamo 

      let xz = miFuncion(4,3);// la funcion es llamada, el valor devuelto sera guardado en la variable xz
      function miFuncion(x,z){
        return x*z; // La funcion retorna el producto de x*z
      }

Acceder a la funcion sin usar los (), nos retornara el objeto de la funcion, en vez del resultado de la funcion

Funciones usadas como valores variables

En JavaScript las funciones son consideradas ciudadanos de primera-clase, lo que signifca que pueden ser tratadas como cualquier otro valor, como una cadena o numero. Esto signifca que las funciones pueden ser usadas de la misma forma en que las variables son usadas para guardar datos.

He aqui 2 ejemplos

      let x = aCelsius(25);
      console.log('La temperatura es '+x+' grados celcius')
      /* 
      en vez de usar una variable para guardar el resultado devuelto de la funcion, podemos usar la funcion 
      directamente como un valor variable 
      */
    
      function multiplicar(a,b){
        return a*b;
      }
      const funcionesMatematicas = {
        funcionDeMultiplicar: multiplicar
      }
      console.log(funcionesMatematicas.funcionDeMultiplicar(10*5));

Las ventajas del segundo ejemplo son que podemos usar la funcion globalmente, ya que si las declaramos directamente dentro del objeto, habria que referenciar al objeto tambien.

Ademas, todo depende de que tan grande sean las funciones, ya que si son pequeñas podriamos ponerlas directamente dentro del objeto, pero si son mas grandes, seria mejor hacer lo que se muestra en el segundo ejemplo y asi hacer nuestro codigo mas legible, organizado y facil de usar

Inclusive, el segundo ejemplo podria ser mejorado usando funciones flecha, ya sea para acortar la sintaxis de la funcion multiplicar o directamente escribirla en el mismo objeto 

      const funcionesMatematicas = {
        multiplicar: (a,b) => a*b;
      }

Que es esto? ¿What is this in js?

En JavaScript la palabra reservada this, hace referencia a un objeto, ahora, a que objeto se refiere, depende de varios factores

Basicamente se refiere a un objeto, pero al objeto al que se refiere depende de como es invocado

  • Cuando se usa this en un metodo dentro de un objeto, this se refiere al mismo objeto
  • Por si solo, this se refiere al objeto global, o sea a window, o a undefined si se usa el modo estricto
  • Dentro de una funcion, this se refiere al objeto global
  • En una funcion en modo estricto, this es indefinido
  • En un evento, this se refiere al elemento que recibio el evento
  • Metodos como call(), apply() y bind() pueden hacer que this se refiera a cualquier objeto

La palabra reservada this

En la definicion de funciones, this se refiere al dueño de la funcion

Como en el ejemplo anterior this se refire al objeto celular que es dueña de la funcion llamar

Es decir que this.marca ser refiere a la propiedad marca de este mismo objeto

Para cuestiones de objetos esto es practicamente simple

Usando metodos built-in o funciones flecha, el scope de this puede cambiar segun como se lo invoque asi que estar atentos a eso y los errores que podria traer
