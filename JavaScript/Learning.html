  • Una ventaja de var es que cuando declaramos con var, se eleva automaticamente al inicio del bloque lo que significa que podemos usar una variable antes de declararla y puede ser inicializada en cualquier momento

Que es esto? ¬øWhat is this in js?

En JavaScript la palabra reservada this, hace referencia a un objeto, ahora, a que objeto se refiere, depende de varios factores

Basicamente se refiere a un objeto, pero al objeto al que se refiere depende de como es invocado

  • Cuando se usa this en un metodo dentro de un objeto, this se refiere al mismo objeto
  • Por si solo, this se refiere al objeto global, o sea a window, o a undefined si se usa el modo estricto
  • Dentro de una funcion, this se refiere al objeto global
  • En una funcion en modo estricto, this es indefinido
  • En un evento, this se refiere al elemento que recibio el evento
  • Metodos como call(), apply() y bind() pueden hacer que this se refiera a cualquier objeto

      var mensaje = "El es el \"elegido\" de la tribu";
      let slash = "Este es el caracter: \\ backslash"

Existen otros 6 caracteres de escape validos en JavaScript

  • \b backspace o borrar
  • \f form feed o alimentar el formulario?
  • \n new linea o nuevo salto de linea
  • \r carriage return o retorno del carro
  • \t horizontal tabulator o tabulacion horizontal
  • \v vertical tabulator o tabulacion vertical

Puede que reconozcas estos caracteres de otros lenguajes, sin embargo no tienen ningun sentido en HTML ya que fueron creados para controlar maquinas de escribir

Separar largas lineas de codigo

Para mejor legibilidad es buena practica dar un nuevo salto de linea si la linea actual supera los 80 caracteres

Si una declaracion de JavaScript no cabe en una linea, lo recomendable es pasar a la siguiente despues de un operador 

      document.getElementById("demo").innerHTML =
      "Hello Dolly!";

Tambien puedes separar una linea de codigo dentro de una cadena usando el caracter \

      document.getElementById("demo").innerHTML = "Hello \
      Dolly!";

Sin embargo el metodo de usar un backslash no es recomendable ya que puede no tener soporte universal ya que algunos navegadores no permiten espacios antes del caracter \

Un mejor manera de separar una cadena es usar concatenacion

      document.getElementById("demo").innerHTML = "Hello"+
      "Dolly!";

Ademas no se puede separar una lina de codigo con un backslash

Cadenas como Objetos

Normalmente las cadenas en JavaScript son valores primitivos creados desde literales

Pero las cadenas tambien pueden ser definidas como objetos usando la palabra clave new

      let ds = new String("JavaScript");

Como ya vimos anteriormente, no se deben crear objetos de tipo cadena

La palabra clave new complica el codigo y ralentiza el tiempo de ejecucion

Ademas objetos de tipo cadena pueden producir resultados inesperados

      let x = 'Joel';
      let y = new String("Joel");
      console.log(x==y);
      //En este caso sera true ya que son iguales en contenido
    
      let x = 'Joel';
      let y = new String("Joel");
      console.log(x===y);
      //Sin embargo aqui sera false ya que son del mismo tipo y por lo tanto no pueden ser comparados de manera segura

Al igual que si comparamos dos objetos cadenas

      let x = new String("David");
      ley z = new String("David");
      console.log(x==z);
      // el resultado siempre sera false ya sea si usamos == o === ya que los objetos no pueden ser comparados entre si

Es mejor dejarlo en primitivos ya que ademas JavaScript convierte momentaneamente las cadenas a objetos cuando usamos sus metodos.

Extrayendo partes de una cadena

Hay 3 formas de extrar una parte de una cadena

Slice()

El metodo slice() nos permite extraer parte de una cadena y nos devuelve la cadena extraida en una nueva cadena

El metodo acepta dos parametros, la posicion inicial y final donde hacer el corte 

      text.slice(7,13);

JavaScript cuenta las posiciones desde 0

Si se omite el segundo parametro, se cortara la cadena hasta el final de esta

Si un parametro es negativo, la posicion es contada desde el final de la cadena

      texto.slice(-14,-7);
      // basicamente se va contando hacia el lado positivo desde la posicion final, es decir -14,-13,etc

Substring()

substring() es similar a slice()

La diferencia radica en que basicamente no podemos introducir valores negativos , ya que estos seran tratados como ceros por el metodo substring()

E igualmente si se omite el segundo parametro, se cortara el resto de la cadena

      libro.substring(5,7);

Substr()

Finalmente tenemos substr()

Es similar a slice(), pero la diferencia es que el segundo parametro dice la cantidad de caracteres que se van a cortar, es decir la longitud de la parte a cortar

Si omitimos el segundo parametro substr() cortara hasta el final de la cadena

Y si el primer parametro es negativo, se empezara a contar desde el final de la cadena

      gg.substr(-2,5);

Reemplazando contenido de una cadena

El metodo replace() reemplaza un valor especificado con otro dentro de una cadena

      let gpt = "Microsoft es una corporacion malevola";
      let google = gpt.replace("Microsoft","Google");
      // google = 'Google es una corporacion malevola';

Cabe resaltar que el metod replace() no cambia la cadena donde es llamado

El metodo replace() retorna una nueva cadena

El metodo solo reemplaza la primera coincidencia

Se puede usar una expresion regular con la bandera global /g puesta para reemplazar todas las coincidencia

Por defecto, replace() es sensible a mayusculas y minusculas

Para que no distinga entre mayusculas y minusculas debemos usar la bandera /i dentro de la expresion regular (i de insensitive) , estas flags son aplicadas al primer argumento del metodo, que se refiere a la parte de la cadena que queremos reemplazar 

      let coca = "me gusta la coca cola y la coca de mascar";
      let papas = coca.replace(/coca/g,"papa");
      // al usar /g, se reemplazan todas las coincidencias
      // las expresiones regulares no van dentro de comillas
    
      let uni = "por favor dale 5 estrellas a Unifranz";
      let umsa = uni.replace(/UNIFRANZ/i,"Umsa");
      // el uso de la bandera /i convierte a replace en insensitivo a mayusculas o minusculas

Ademas replaceAll() nos deja poner como en replace() una expresion regular en lugar de una cadena, aunque en este caso siempre debemos usar la bandera global /g para evitar errores.

Concatenacion

Usando el metodo concat(), se nos permite unir 2 o mas cadenas y parece ser un buen reemplazo a +, ya que concat() nos permite unir infinitas cadenas y + solo 2 a la vez 

      let palabra1 = "Hola";
      let palabra2 = " Pianola";
      let miTexto = palabra1.concat(palabra2);
      // Hola Pianola
    
      text = "Hello" + " World!";
      text = "Hello".concat("World!");

Podrias decir que + es mas facil de escribir, pero la ventaja de concat() es que siempre retorna una nueva cadena y que ademas puedes concatenar cuantas cadenas como quieras

Consideraciones en cadenas

Como vimos, todos los metodos retornan una nueva cadena y no modifican la cadena original, es decir, las cadenas son inmutables, las cadenas no pueden ser cambiadas, solo reemplazadas

JavaScript trim(), trimStart() y trimEnd()

El metodo trim() remueve los espacios en ambos lados de una cadena

Los metodos trimStart() y trimEnd() fueron agregados en ES2019 y pues hacen lo que dice su nombre no?, remueven espacios al inicio y final de una cadena respectivamente

Si quisieramos reemplzar los espacios entre palabras, podriamos usar replaceAll()

      let str = "PB 10 CV 2662";
      str = str.replaceAll(' ', '');

Rellenado de Cadenas en JavaScript

ES2017 agrego dos nuevos metodos para poder agregar "relleno" a las cadenas

El relleno basicamente concatena la cadena original con otra cadena que le digamos, hasta que la cadena sea del valor que igual pusimos dentro del metodo, si la cadena ya es de ese tama√±o o mas grande, entonces no se agrega nada, y si no se especifica el segundo parametro, el valor por defecto es espacio.

padStart()

padStart() como indica su nombre, nos permite rellenar una cadena con otra que le digamos hasta que la longitud deseada sea alcanzada 

      let cadena = "XD";
      let nuevaCadena = cadena.padStart(5,"X");
      // XXXXD

padStart() es un metodo de cadenas, para usar numeros, antes debemos convertir el numero a una cadena 

      let numerito = 25;
      let cadenita = numerito.toString();
      let relleno = cadenita.padStart(4,"0");
      // 0025

padStart() y padEnd() son caracteristicas de ES2017 por lo tanto no funcionan en navegadores previos a esa version

padEnd()

padEnd() es exactamente igual que padStart() solo que el relleno se pone al final de la cadena 

      let txt = "cadena";
      let rellenada = txt.padEnd(7,"s");
      //cadenas

Convirtiendo una cadena en un arreglo

JavaScript split()

Una cadena puede ser convertida a un arreglo con el metodo split()

      texto.split(",")
      texto.split(" ")
      texto.split("|")

El argumento que acepta, son los separadores

Si el separador es omitido, el arreglo devuelto contendra toda la caden en la posicion 0 del arreglo

split() funciona de la siguiente manera

Dentro del metodo podemos poner un parametro donde se recibe el separador cabiendo resaltar que la cadema a separar debe contener por lo menos un caracter igual al que usamos de separador, es decir, que si tenemos en nuestra cadena original varias comas, podemos elegir usar las comas como separador al convertir la cadena en un arreglo. Basicamente el parametro que acepta split() define el separador, ya sea espacios, comas, puntos o incluso una misma letra

Busqueda dentro de cadenas

JavaScript indexOf() y lastIndexOf()

El metodo indexOf() retorna la posicion de la primera coincidencia especificada a buscar dentro de una cadena, indexOf se traduciria como indiceDe() 

      var texto = 'encuentra la posicion de "encuentra"';
      var posicion = texto.indexOf("posicion");

Como ya sabemos, JS cuenta posiciones desde 0

lastIndexOf() por otra parte retorna la posicion de la ultima ocurrencia especificada a buscar dentro de una cadena

Ambos indexOf() y lastIndexOf() retornan -1 si es texto a buscar no es encontrado

Ambos metodos aceptan un segundo parametro que indica desde que posicion empezar la busqueda

En el caso de lastIndexOf() se cuenta al reves, es decir, si se le especifica una posicion de donde empezar la busqueda, entonces el metodo buscara desde esa posicion hasta el inicio de la cadena

JavaScript search()

El metodo search() busca una cadena(o expresion regular) dentro de una cadena y retorna la posicion de la coincidencia 

      let teseracto = 'Glorious Purpose!!!';
      let loki = teseracto.search("Glorious")
      let lokiRegex = teseracto.search(/Glorious/);

La diferencia entre search() e indexOf() es que search acepta expresiones regulares, pero no puede indicar una posicion desde donde emprezar a buscar, lo cual indexOf si puede hacer pero no puede usar expresiones regulares.

JavaScript match()

El metodo match() devuelve un arreglo que contiene todas las coincidencias resultantes al buscar una cadena dentro de otra cadena

      let texto = "Pablito clavo un clavito en la calva de un calvito";
      texto.match("ito");

Tambien podemos usar expresiones regulares dentro de match y cabe resaltar que match por si solo solo devuelve la primera coincidencia , si queremos que devuelva todas las coincidencias, debemos usar al igual que en los metodos anteriores, la bandera global, /g, e i/ si queremos que no distinga entre mayusculas y minusculas

      let texto = "Pablito clavo un clavito en la calva de un calvito";
      texto.match(/ito/g); //global sensible
      texto.match(/ito/gi); //global insensible

JavaScript matchAll()

Introducido en ES2020, matchAll() hace lo que dice, devuelve todas las cadenas que coinciden con la busqueda en un arreglo, y al igual que replaceAll() acepta expresiones regulares, y la bandera global /g es obligatoria

      let pablo = "Pablito clavo un clavito en la calva de un calvito";
      let objetoIterable = pablo.matchAll("ito");
      //TODO: verificar y comprender que es un iterador en JavaScript

matchAll() devuelve un iterador, no un arreglo

JavaScript includes()

El metodo includes() (incluye) retorna true si una cadena contiene un valor especificado ademas tiene un segundo parametro que permite elegir la posicion desde la cual se empezara a buscar, y al igual que en otros metodos, se puede usar numeros negativos para iniciar la busqueda al rever

      let text = "Hello world, welcome to the universe.";
      text.includes("world", 12);

includes() es case sensitive, y fue agregado en ES6

JavaScript startsWith() y endsWith()

Estos metodos retornan true si una cadena empieza o termina con un valor especificado respectivamente

Al igual que el anterior metodo, tienen un segundo parametro para indicar desde donde buscar en el caso de startsWith() y hasta donde buscar en el caso de endsWith()

Igualmente son case sensitive y fueron agregados en ES6

      let hola = "Hola Mundo Hola mundo mundoso";
      let c1 = hola.startsWith("Hola");
      let c2 = hola.startsWith("Hola",11);
      let x1 = hola.endsWith("mundoso");
      let x2 = hola.endsWith("mundo,20");
